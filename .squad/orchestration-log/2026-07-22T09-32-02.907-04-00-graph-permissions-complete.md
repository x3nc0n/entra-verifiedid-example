## Reusable pattern

For future live Graph admin operations in this repo that must run non-interactively in a headless shell, prefer:

1. Sign in once with Azure CLI as the needed tenant admin.
2. Acquire a delegated Graph token with:
   - `az account get-access-token --resource https://graph.microsoft.com --tenant <tenant-id>`
3. Convert that token to a secure string and connect with:
   - `Connect-MgGraph -AccessToken $secureToken`

This pattern successfully reused the already-authenticated `az` CLI session as `admin@spaidfamily.onmicrosoft.com` and avoided the unreliable `Connect-MgGraph -UseDeviceAuthentication` path, which crashed in this automation shell on later cmdlet calls with `DeviceCodeCredential authentication failed: Object reference not set to an instance of an object`.
